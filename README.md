# wrtun

Утилита для туннелирования трафика через WebRTC с поддержкой SOCKS5. Работает в клиент-серверной архитектуре: клиент и сервер общаются по протоколу SOCKS5 поверх WebRTC.

## Текущее состояние

**Статус проекта**: Разработка приостановлена. Базовая функциональность туннелирования работает, но проект не завершён.
Проект https://github.com/cacggghp/vk-turn-proxy и множество похожих были созданы во время написания мною этой утилиты.

## Архитектура

- **Клиент**: Слушает SOCKS5-подключения локально и туннелирует их через WebRTC
- **Сервер**: Получает SOCKS5-запросы через WebRTC и отправляет ответы обратно
- **Интерфейс Transport**: Подключаемый, общий для клиента и сервера, транспорт для обмена signaling сообщениями и установки WebRTC соединения
- **Интерфейс Provider**: Подключаемый провайдер для получения STUN/TURN серверов

## Реализованные возможности

- Туннелирование SOCKS5-трафика (пока только TCP) через WebRTC data channels
- Клиент-серверная WebRTC-negotiation (Vanilla ICE) с произвольными транспортами для обмена signaling сообщениями
- Логирование статистики ICE (RTT, типы кандидатов)
- Автоматическое получение публичных STUN серверов из репозитория https://github.com/pradt2/always-online-stun или приватных TURN серверов мессенджера MAX
- CLI с использованием cobra/viper

## Компоненты

### Туннель (`pkg/tunnel/socks5`)
- Интеграция SOCKS5-сервера через `things-go/go-socks5`
- Двунаправленное копирование данных между TCP и WebRTC data channel
- Логирование ICE/RTT статистики

### WebRTC (`pkg/wrtc`)
- Listener для входящих data channel (серверная сторона)
- Negotiation соединений (offer/answer)
- Настраиваемый SCTP буфер приёма (по умолчанию 32MB)

### Провайдеры
- **Public STUN**: Получение STUN серверов из удалённых текстовых списков 
- **MAX**: Использование MAX Chat API для получения ICE серверов и сигналинга

### Транспорты
- **MAX**: Сигналинг через сообщения MAX Chat (base64-кодирование)

## Использование CLI

```bash
# Запуск сервера
wrtun server --transport max --provider max

# Запуск клиента (SOCKS5 на localhost:7331)
wrtun client --transport max --provider max --port 7331
